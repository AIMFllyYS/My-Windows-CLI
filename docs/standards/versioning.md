# 版本号规范

本项目遵循语义化版本（Semantic Versioning）原则，格式为 `MAJOR.MINOR.PATCH`。

## 版本号规则

| 变更类型 | 版本位 | 示例 | 说明 |
|---------|--------|------|------|
| 补丁修复 / 小改动 | PATCH (`0.x.Y`) | `0.1.0` → `0.1.1` | Bug 修复、文档更新、代码优化 |
| 功能更新 | MINOR (`0.X.y`) | `0.1.1` → `0.2.0` | 新增功能模块、重要重构 |
| 核心突跃 | MAJOR (`Z.x.y`) | `0.9.0` → `1.0.0` | 架构级变更、里程碑版本、不兼容变更 |

## 当前阶段

项目处于 `0.x.y` 阶段，表示仍在快速迭代中。MAJOR 版本升级（1.0.0）需满足：

- 核心功能完整且稳定
- API / CLI 接口不再频繁变动
- 经过充分测试

## PATCH 升级时机

- 修复已有功能的 Bug
- 文档补充或修正
- 代码风格 / 性能优化（不影响功能）

## MINOR 升级时机

- 新增功能模块（如新增 `--chat` 模式）
- 重要重构（如模块化拆分）
- 新增 CLI 选项

## MAJOR 升级时机

- 架构级重写
- CLI 接口不兼容变更
- 核心依赖重大升级
